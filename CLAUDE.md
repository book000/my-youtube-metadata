# CLAUDE.md

## 目的
このドキュメントは、Claude Code の作業方針とプロジェクト固有ルールを示します。

## 判断記録のルール
- 判断内容の要約
- 検討した代替案
- 採用しなかった案とその理由
- 前提条件・仮定・不確実性
- 他エージェントによるレビュー可否

## プロジェクト概要
- 目的: YouTube 動画のメタデータ管理
- 主な機能: `tracks.json` によるデータ保持

## 重要ルール
- 会話言語: 日本語
- コミット規約: Conventional Commits
- コメント言語: 日本語
- エラーメッセージ言語: 英語

## 環境のルール
- ブランチ命名: Conventional Branch (`feat/`, `fix/` 等)
- GitHub リポジトリ調査: テンポラリディレクトリに clone して検索
- Renovate PR: 追加コミット不可

## Git Worktree
- 使用していません。

## コード改修時のルール
- エラーメッセージの絵文字統一 (既存にあれば)
- TypeScript: `skipLibCheck` 禁止 (該当なし)
- docstring: 日本語で記載 (該当なし)

## 相談ルール
- Codex CLI: 実装レビュー、局所設計、整合性確認
- Gemini CLI: 外部仕様、最新情報確認
- 指摘への対応ルール: 黙殺禁止

## 開発コマンド
- パッケージマネージャー: なし
- ビルド/テスト: なし
- `tracks.json` の編集が主作業

## アーキテクチャと主要ファイル
- `tracks.json`: メタデータ定義ファイル

## 実装パターン
- JSON オブジェクトのキーは YouTube Video ID

## テスト
- 自動テストなし
- JSON 構文確認必須

## ドキュメント更新ルール
- 変更があった場合、適宜 README 等を確認 (現在はなし)

## 作業チェックリスト
- 新規改修時: 最新の master からブランチ作成
- コミット前: Lint (JSON Lint 推奨) 確認
- PR 作成前: コンフリクト確認
- PR 作成後: CI 確認 (あれば)

## リポジトリ固有
- データ整合性を重視してください。
